Dipshikha Krishi Sahakari Sanstha Ltd.
Version 1.0 | Nepal Context
1. Executive Summary
Dipshikha Krishi Sahakari Sanstha Ltd. is transitioning to a modern, digital-first infrastructure.
This platform integrates a high-end public website, a secure Member Portal, and a
comprehensive Dynamic Admin Panel. The system is designed to provide financial
transparency, market accessibility, and operational efficiency for the cooperative staff and its
farmer members.
2. Technical Architecture
Frontend Next.js 14+ (App Router), TypeScript, Tailwind CSS
UI / Framework ShadCN UI, Framer Motion, Radix UI
Backend / Auth Supabase (PostgreSQL, Realtime DB, Auth, Storage)
Data Access Drizzle ORM, TanStack Query (React Query)
SMS Gateway Sparrow SMS (API for OTP & Member Alerts)
Deployment Vercel (Production)
3. Design System (Dipshikha Aesthetic)

5. Dynamic Admin Control Panel
The Admin Panel is a secure, role-based dashboard for cooperative staff to manage all aspects
of the platform without code.
5.1 Content Management (CMS):
Notices & News:  Post urgent alerts, vacancies, and news updates to the homepage
ticker and blog.
Gallery & Media:  Upload and categorize photos of field visits, training sessions, and
events.
Downloads:  Manage audit reports, bylaws, and membership forms.
5.2 Operational Controls:
Member Approval:  Review pending KYC applications, verify documents, and send
auto-approval SMS.
Market Price Manager:  Update daily crop prices that sync instantly to the public
website.
SMS Broadcast:  Send bulk notifications to all members or specific groups.•

6. Implementation Roadmap
Phase Objective Key Deliverables
Phase 1 Public Presence Landing page, News/Notices CMS, Team/Gallery, SEO setup.
Phase 2 Onboarding Member Registration, KYC Upload, Admin Review Workflow, SMS OTP.
Phase 3 Operations Member Dashboard, Financial Sync, Market Watch, Advisory Desk.
7. Security & Privacy
Data Privacy:  Use of Supabase Row Level Security (RLS) ensuring members only
access their own financial records.
Audit Logs:  All admin actions (price updates, document approvals) are logged with
timestamps and user IDs.
Storage:  Encrypted buckets for sensitive KYC documents with signed URL access.
